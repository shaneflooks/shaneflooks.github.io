Step by step instruction on Proactive Outreach Demo

Admin Section

  1. Open OneCloud Ticket with Cloud Ops to have POM enabled
  2. Administrator -> Account Settings
    • Ensure that Proactive Outreach Preview Dialing is available under unified communication services
    • Check Voice Channel to validate if there is a Proactive Outreach number and Display Name Present
  3. Administration -> Workspaces -> Layout Manager
    • Clone Default Account Layout saving it as Proactive Outreach Layout
    • Administration -> Workspaces -> Layout Manager -> Proactive Outreach Layout- Welcome Page -> Edit Edit the profile -> Tabs -> Welcome Page
    • Select Split Pane--Select a Layout--> Pane 1 and 2
    • Select Widgets
      • Widget 1-->Outbound Connector
      • Widget 2-->Leave as Welcome Page Save
    • Views--Custom-->Edit Custom
    • Select a Layout--> Pane 1 and 2 --> Select Split Pane
    • Select Widgets
      • Widget 1-->Outbound Customer Details
      • Widget 2--> Outbound Tools -Save
    • -Add a new Tab named Outbound Agent Script
    • -Role-->Agent and Supervisors
    • -Icon-->Neo-Icon Screen pop
    • -Select a Layout--> Pane 1--> Single Pane
    • -Widget 1--> Outbound Agent Script Screen Pop
    • -Save
  4. Create a new UX Profile for the associated layout
    • -Clone default to create
    • -Name it Proactive Outreach UX Profile
    • -Select Layout-->Proactive Outreach Layout
    • -Save
  5. Add queues as required for Proactive Outreach
    • -Save
  6. Navigate to Contact Center -> Create a new Category Named System with an Attribute Outbound
    • -Save
  7. Create a context Data Rule in Administration
    • -Business Rules -> Context Data Rule -> Add a Rule
    • -Rule Name--Proactive Outreach
    • -Add Key Value--> queueName
    • -Value = Queue Name This is case sensitive
  8. Queue Configuration
    • -Routing Queue -> Select Queue from Drop Down
    • -Routing Attributes -> System Outbound
    • -Orchestration Flow -> Onecloud Proactive Outreach Acquire
    • -Save
  9. Administration -> Channels -> Voice
    • -Add Routing Attributes -> System Outbound
    • -Enable Apply Context Data Matching
    • -Add Context Data Rule named Proactive Outreach
  10. Create a Proactive outreach profile to administer the feature.
    • -Administrator -> User Management -> Profiles
    • -Create a Proactive Outreach Profile
  11. Assigning Required Queues
    • -Membership = Agent
    • -Matching Attributes = SystemOutbound
    • -Add Queue
    • -Save
  12. Add users to newly created profile that require proactive outreach.

Agent Assist AI generated summary

Get quick access to information and other resources during an interaction from the Agent Assist widget. The widget provides an AI generated summary of the conversation and helps you to reply quickly and accurately to the customer using online resources. Agent Assist widget

After Contact Work

Perform additional work and review the interaction details after an interaction. In the After Contact Work (ACW) state, you can also view the interaction transcript. If the account administrator enables After Contact Work, you automatically enter the ACW state after the interaction ends. After Contact Work

Consult interactions

Consult an agent or a supervisor before completing a customer query. After the consultation, you can complete the consult call as a transfer or as a conference. Consult interactions

Corporate contacts overview

Use the Corporate Contacts widget to view Avaya Cloud Office™, Avaya Spaces, and Microsoft Teams contacts and contact groups, make direct and consultation calls to contacts, transfer customer calls, and add contacts to the Favorites list. For the Avaya Cloud Office™ and Microsoft Teams providers, you can also send messages to contacts and chat groups and participate in conversations. Using corporate contacts in Avaya Workspaces

Customer Journey widget

Use the Customer Journey widget to have a visual representation of customer’s interactions. A complete view of the customer journey across all channels can help to facilitate informed decisions and provide a better customer service. Agents can use the interaction search functionality to view past customer journeys. The interaction search enables agents to filter by activities, date, channels, and identifiers. After the search, agents can choose the customer journey to review from a list of matches. Agents can also review the relevant transcripts of the interactions. Customer Journey widget Viewing offline interactions

Digital Library widgets

Use the Digital Library widget to respond to interactions quickly with global and queue templates, such as email and quick text templates. Digital Library widgets

Disposition Codes

Select disposition codes for active voice, email, chat, and messaging interactions. The account administrator configures disposition codes, and you can use them to report the outcome of an interaction. Working on a voice interaction Working on an email interaction Working on a chat interaction Working on a messaging interaction

External calling and outbound emails

Call an external number, including PSTN and SIP numbers. You can also send outbound emails during an active customer interaction on the email channel. Making an external call Adhoc email

Not Ready reason codes

In the Agent Details pane, select a reason code for changing your state to Not Ready. You can view the selected reason code next to your state. The Not Ready reason codes also appear in the real-time and historical reports that are available to supervisors. The account administrator configures the reason codes for going into the Not Ready state in Application Center Administration. Agent states

Screenpops

Access external web pages during an active voice, email, chat, or messaging interaction. For example, you can access an external website with the information about current currency exchange rates. The account administrator enables and configures access to external web pages. Working on a voice interaction Working on an email interaction Working on a chat interaction Working on a messaging interaction

Transfer

Send an interaction to another queue or directly to an agent or a supervisor. You can also transfer a voice interaction to a number outside of the contact center. Transferring an interaction to a queue Transferring an interaction to a user Transferring a voice interaction to an external number

Real-time reporting

As a user with the Reporting Supervisor role, you can access the Realtime Reporting dashboard to run real-time reports. Using the dashboard, you can monitor up-to-date statistics for your contact center and resources. Real-time reporting overview

Supervisor Features - Barge-in

As a supervisor, barge into an interaction to help close a sale or interact with the customer. To barge in, you must first observe or coach the interaction. This feature is available only for chat, messaging, and voice interactions Barging into an interaction

Supervisor Features - Coach

As a supervisor, coach an agent using the Coach feature. To coach, you must first observe the interaction. This feature is available only for chat, messaging, and voice interactions. Coaching an agent

Supervisor Features - Forced agent logout and state change

As a supervisor, change an agent state from Ready to Not Ready to prevent agents from receiving interactions. You can also forcibly log an agent out of Avaya Workspaces and end their session. Agent state transitions

Supervisor Features - Observe

As a supervisor, monitor ongoing interactions between an agent and a customer to assess whether intervention is required. This feature is available only for chat, messaging, and voice interactions. Observing an interaction
